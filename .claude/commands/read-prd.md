# Read PRD Document

Read a specific PRD document from docs/prd/ and provide a summary of what needs to be built.

Read the specified PRD document completely and output:
1. A brief summary of the component
2. Key dependencies (what must be built first)
3. File manifest (all files to create)
4. Environment variables needed
5. Acceptance criteria

Usage: /read-prd <doc-number>
Example: /read-prd 03 (reads 03-SERVER-CORE.md)
